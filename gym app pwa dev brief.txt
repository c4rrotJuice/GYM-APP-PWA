Gym Management PWA — Complete Development Brief

the project is best positioned as a mobile-first Progressive Web App (PWA) focused on gym operations, attendance automation, member retention, and lightweight business management.

The system should prioritise:

operational simplicity,
cross-device access,
low maintenance,
offline resilience,
and future SaaS scalability.

Core stack direction:

Frontend: Vanilla HTML/CSS/JavaScript
Backend services: Supabase
Hosting: Netlify
Database: PostgreSQL (via Supabase)
Authentication: Supabase Auth
Deployment model: Static PWA + cloud backend
1. PROJECT OVERVIEW (Executive Summary)
Project Name

Gym Management Progressive Web App (Gym PWA)

Purpose

Develop a full-featured gym management system that:

helps gym owners manage operations,
automates attendance,
tracks memberships/payments,
improves member retention,
and provides members with a convenient mobile-first experience.

The application will function as:

an installable PWA,
a cross-device synced system,
and a lightweight SaaS-ready architecture.

2.5 Attendance System (Critical Feature)
Attendance Method

Attendance will use:

dynamic QR code scanning.
Attendance Flow
Admin generates attendance QR.
QR validity is configured:
Weekly
Fortnightly
Monthly
QR is displayed physically inside the gym.
Member opens installed PWA.
Member scans QR.
App validates:
membership status,
QR validity,
user role.
Attendance entry is logged.
2.6 QR System Rules
QR Must NOT Contain
passwords,
payment data,
sensitive information.
QR Should Contain
gym identifier,
attendance token,
expiry metadata,
validation signature.
2.7 QR Validity Modes
Type	Validity

Fortnight	14 days
Monthly	30 days
2.8 Attendance Validation Rules

Attendance succeeds only if:
